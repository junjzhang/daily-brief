Dynamo Isolate Recompiles for torch.compile
Dynamo 2026-05-04 · Xiao Fu, William Wen, Animesh Jain, Laith Sakka
技术要点
  • 新增 isolate_recompiles=True 参数，为每个 torch.compile() 调用提供独立的缓存 bucket，解决多 wrapper 共享同一底层 code object 时的缓存碰撞问题。
  • 缓存结构由原来的单链表（per code object）升级为 unordered_map<int64_t, list<CacheEntry>>，bucket -1 保持向后兼容，bucket ≥ 0 分配给各 isolated 实例。
  • 查找时优先搜索自己的 bucket，若未命中则以只读方式回退到默认 bucket，避免跨 wrapper 污染。
  • 每个 torch.compile 调用现在可独立设置 recompile_limit，不再共享全局 8 次上限。
  • Factory 模式及 static/dynamic 混合编译场景均得到原生支持，无需 f.__code__.replace() 等 workaround。

Printing and Inspecting Tensors Inside torch.compile
Dynamo 2026-05-06 · Xiao Fu, Shangdi Yu
技术要点
  • torch._higher_order_ops.print 支持在不触发 graph break 的情况下打印张量，兼容 DTensor 并可按 rank 过滤输出。
  • @leaf_function 装饰器使 Python 函数对编译器不透明，可执行文件写入、范数计算、张量哈希等自定义操作而不产生 graph break。
  • debug_grad_log 提供一行式梯度范数日志，零样板代码即可在前向/反向 pass 中捕获梯度信息。
  • install_debug_prints 可自动为整个 nn.Module 注入日志，同时捕获前向激活值和反向梯度。
  • 所有工具均无 graph break、支持 DTensor、可同时用于前向与反向 pass——提供系统性调试框架而非临时 hack。

torch.compile for TorchTitan RL: 6x Faster Unified RL Training
Distributed 2026-05-06 · Lucas Kabela
技术要点
  • TorchTitan RL 采用统一模型定义，Trainer (TorchTitan) 与 Generator (vLLM) 共享同一份模型权重，避免竞争框架中双份模型定义的开销。
  • 在 Qwen3 0.6B / GSM8K（TP=4，8×H100）上，端到端训练时间从 446 秒降至 70.4 秒，整体提速约 6×。
  • Generator 提速 48×（262.4s → 5.4s），Trainer 提速 9×（157s → 17.1s），共享编译额外节省 36 秒启动开销。
  • 关键实现：DTensor CUDAGraph 支持、变长序列 packing、统一 full-graph 捕获与共享编译产物、位级确定性数值对齐。
  • 主要挑战是 vLLM 互操作性及特定 DTensor 操作中的弱引用生命周期管理（cudagraph 生命周期）。
